# X-EAS: Academic Residue batch generator in docx format

## 简介
写论文时最痛苦的是什么？不是内容，是格式。页眉页脚怎么调都不对，行距字号改了又改，参考文献格式更是噩梦——更可怕的是，每个老师的要求都不一样，每次都要从头再来。本项目旨在终结这种低效重复劳动：只需提供一份标准模板，工具即可自动解析其中的格式规则，并将这些样式精准应用到你的文档内容上，实现"内容不变，一键换装"。

## 适用场景
- 课程论文: 快速匹配老师给的模板
- 毕业设计: 符合学校官方格式要求
- 实验报告: 统一实验室文档规范
- 竞赛文档: 满足提交格式标准

## 使用示例

### 基本用法

**语法格式：**
```
参考 [模板文件路径] 的docx模板，编写相同格式(要求所有格式全部相同，包括页眉页边距，段落格式等所有东西)但内容不同的，标题为：[新文档标题]
```

### 参数说明

- `[模板文件路径]`: 你的Word模板文件的完整路径
- `[新文档标题]`: 你想要生成的新文档的标题

### 具体示例

**示例1：生成毕业设计任务书**
```
参考 test_cases/Southwest Jiaotong University Thesis Template/任务书.docx 该任务书的docx模板，编写相同格式(要求所有格式全部相同，包括页眉页边距，段落格式等所有东西)但内容不同的，标题为：基于ESP32的无感FOC驱动设计的任务书
```

**示例2：格式转换**
```
将 test_cases/正文内容.md 按照 test_cases/Southwest Jiaotong University Thesis Template/正文.docx的格式转换为Word文档。保证所有的格式的一致
```


### 使用步骤

1. 准备一个格式正确的Word模板文件（.docx格式）
2. 按照上述格式构造指令
3. 工具会自动：
   - 读取模板文件的所有格式设置
   - 保持原有的页眉页脚、页边距、段落格式等
   - 将新内容应用到文档中
   - 生成格式完全一致但内容不同的新文档

### 注意事项

- 模板文件必须是.docx格式
- 确保模板路径正确且文件存在
- 新标题会自动替换模板中的原有标题
- 所有格式（字体、行距、缩进、页眉页脚等）都会保持不变
