# 分段检索式改写流程计划

## 一句话目标
把现有流程从“按字数切块后直接改写”，升级为“先理解模板结构，再按语义切块，再从用户素材里检索相关内容后改写”。

## 核心原则
- 模板和素材都先按段落编号，后续所有处理都围绕编号进行。
- 先做结构理解，再做切块，再做检索，最后再改写。
- Python 负责提取、组装、校验和还原。
- LLM 负责分段理解、相关素材筛选和逐块改写。

## 整体流程

### 1. 提取模板全文
- Python 从模板 `docx` 中提取正文，生成 `full_paragraphs`。
- 每段保留唯一的 `[PARA_ID]`，作为后续分段、改写、校验、还原的锚点。
- 输出形式建议为“每行一段”的纯文本。

### 2. 识别模板的语义分段
- 将 `full_paragraphs` 按滑动窗口分批送给 LLM。
- 提示词模板输入：当前窗口内的 `[PARA_ID] + 段落文本`、分段规则、必要的前后文信息。
- LLM输出：模板 `segment` JSON 列表，每项包含 `segment_id`、`summary`、`purpose`、`para_ids`。
- 示例 JSON：
```json
[
  {
    "segment_id": "seg_1",
    "summary": "介绍研究背景与问题定义",
    "purpose": "background",
    "para_ids": ["A1B2C3D4", "A1B2C3D5", "A1B2C3D6"]
  }
]
```

### 3. 校验模板分段结果

- Python 校验分段结果是否可靠：不能丢失 `para_id`，不能重复，顺序不能打乱，必须覆盖全文。
- 如果校验失败，就重新分批处理或回退到人工检查。

### 4. 生成改写用的 chunk

- 先按 `segment` 切分，这是第一优先级。
- 如果某个 `segment` 太大，再在该 `segment` 内按字数或自然断点继续拆分。
- 这样得到的 `chunk` 同时保留语义完整性和长度可控性。

### 5. 处理用户素材

- 用户上传的 `txt/docx/md` 先按段落或换行切开，并编号为 `[1] [2] [3] ...`。
- 再把素材全文按与模板相同的方式送给 LLM 做语义分段。
- 输出结构同样是 `segment -> para_ids`，但这里的 `para_ids` 对应素材编号。
- 提示词模板输入：编号后的素材全文、分段规则、必要的前后文信息。
- LLM输出：素材 `segment` JSON 列表，每项包含 `segment_id`、`summary`、`para_ids`。
- 示例 JSON：
```json
[
  {
    "segment_id": "mseg_1",
    "summary": "介绍FOC控制原理",
    "para_ids": ["1", "2", "3"]
  }
]
```

### 6. 为每个 chunk 检索相关素材

- 对每个模板 `chunk`，把当前 `chunk` 的 `segment` 信息和所有素材的 `segment` 信息一起交给 LLM。
- 这里的目标不是直接改写，而是先缩小可用素材范围。
- 提示词模板输入：当前 `chunk` 原文、当前 `chunk` 所属 `segment`、全部素材 `segment`、用户写作任务。
- LLM输出：相关素材 `para_ids`、当前 `chunk` 的写作大纲，必要时附匹配理由。
- 示例 JSON：
```json
{
  "material_para_ids": ["4", "7"],
  "outline": "先说明研究背景，再引出控制方法",
  "reason": "这两段和当前 chunk 主题最接近"
}
```

### 7. 组装逐块改写任务

- Python 为每个 `chunk` 组装一个独立任务包，至少包含当前 `chunk` 的大纲、原文和检索出的相关素材内容。
- 然后把任务包发给 LLM 做逐块改写。
- 提示词模板输入：当前 `chunk` 原文、当前 `chunk` 大纲、检索出的素材原文、改写约束。
- LLM输出：改写后的 `chunk`，要求保留 `[PARA_ID]`、行级结构和必要格式标签。

### 8. 校验并还原结果

- Python 校验改写结果是否满足约束：行数不变，`[PARA_ID]` 不变，格式标签不变，公式结构合法。
- 校验通过后，再按现有流程还原为新的 `docx`。

## 这套方案解决的问题

- 避免只按字数切块，导致语义被打断。
- 避免整份素材一起喂给模型，造成上下文浪费。
- 让每个改写块都能拿到更相关的素材和更清晰的写作目标。
- 提高改写的一致性、可控性和可校验性。
