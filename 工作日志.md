# 工作日志

## docx-essay-writer Skill 开发总结

### 项目概述

创建了独立的 `docx-essay-writer` skill，实现四阶段工作流：**模板分析 → AI 内容生成(Markdown) → 用户审阅 → 格式转换+验证**。所有脚本独立实现，不依赖其他 skill。

核心规则：参考文献禁止编造（Rule 0）、必须先分析模板（Rule 1）、章节结构与模板一一对应（Rule 2）、参考文献须用内置检索工具（Rule 3）。

### 模块清单

| 脚本 | 功能 |
|------|------|
| `analyze_template.py` | 解析 DOCX 模板：标题层级树、样式映射、页面布局、段落格式、特殊元素统计 |
| `search_references.py` | 调用 OpenAlex API 检索真实学术文献，输出 GB/T 7714-2015 格式，支持多主题去重 |
| `md_to_docx.py` | Markdown → DOCX 转换，样式/布局全部从模板动态读取 |
| `verify_format.py` | 对比输出与模板的格式一致性（页边距、页面大小、页眉页脚、样式文件、行距） |

### md_to_docx 核心能力

- **动态样式**：从 `styles.xml` 按名称查找 styleId，优先级链 `论文正文 → Body Text → Normal`
- **页面布局**：从模板 `sectPr` 直接复制，不硬编码
- **LaTeX → OMML**：递归解析 12 级优先级（函数括号 → 矩阵 → cases → 根号 → 分数 → 上下标 → 符号）
- **支持元素**：标题(h1-h3)、正文、加粗、行内/块级公式、表格、图片、有序/无序列表、代码块

### v2 重构：消除硬编码

**核心改动**：新增 `extract_template_formats()` 函数，从模板动态提取所有格式参数。

| 修复项 | 修复前(硬编码) | 修复后(动态提取) |
|--------|---------------|-----------------|
| 正文样式 | `Body Text` (id=5) | `论文正文` (id=29)，频率分析自动识别 |
| 标题 numPr | 固定 `ilvl='1', numId='0'` | 按模板各级标题样式读取，无则不添加 |
| 公式字体 | `eastAsia='宋体'` | 样式链追踪（style → link → basedOn） |
| 语言设置 | `en-US / zh-CN` | 从样式链动态提取 |
| 列表缩进 | `left/hanging='420'` | 从模板列表/正文样式提取 |
| 图片最大宽度 | `5274310` EMU | `pgSz.w - pgMar.left - pgMar.right` 动态计算 |
| font_hint | 全部 `'eastAsia'` | 从 `docDefaults` 推导 |

新增专用样式识别：参考文献(id=47)、图片(id=39)、三线表(id=46)。

### v3 修复：三线表支持

**问题**：表格使用全边框，而模板通过单元格级 `tcBorders` 实现三线表（表头上粗线+下细线，末行下粗线，其余无边框）。

**方案**：`extract_template_formats` 新增单元格级边框提取（header/body/last 三种模式），`_add_table_row` 按行位置应用对应边框、底纹、缩进、文字属性。

### 测试验证

两项测试均通过完整格式验证（页边距、页面大小、页眉页脚、styles.xml 完全匹配）：

1. **任务书**：XML 文本替换方式，表单式文档格式完全一致
2. **绪论**：Markdown → DOCX 转换，含 30 个 OMML 公式、7 个表格、10 篇真实参考文献
