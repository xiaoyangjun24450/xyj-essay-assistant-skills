# 工作日志

## 项目概述

**项目名称**: X-EAS: Academic Residue Batch Generator in DOCX Format (docx-essay-writer)

**项目简介**: 专业的 Word 文档论文生成工具,能够从 DOCX 模板提取所有格式信息(字体、字号、图片、表格、页眉页脚等),将新内容完美还原到模板中,实现"内容不变,一键换装"。

**适用场景**:
- 课程论文: 快速匹配老师给的模板
- 毕业设计: 符合学校官方格式要求
- 实验报告: 统一实验室文档规范
- 竞赛文档: 满足提交格式标准

## 技术路线说明

### 核心原理

本项目采用"提取-编辑-还原"的三阶段工作流程:

#### 1. 提取阶段 (extract_docx.py)
- 读取 DOCX 模板文件
- 遍历所有段落(paragraph)和表格(table)
- 提取每个文本段(run)的内容和格式信息
- 生成带有位置标记和格式信息的 Markdown 文件
- 支持嵌套表格、图片、域代码等复杂元素

#### 2. 编辑阶段
- 在 Markdown 文件中编辑文本内容
- 保留所有位置标记(如 `<!-- PARA:10,RUN:3,FORMAT:... -->`)
- 参考文献部分使用 OpenAlex API 搜索真实学术文献
- 使用内置工具 `search_references.py` 避免编造参考文献

#### 3. 还原阶段 (restore_docx.py)
- 复制原始 DOCX 模板作为基础
- 根据 Markdown 中的位置标记替换对应文本段的内容
- 保留所有原始格式、图片、表格结构
- 生成格式完全一致但内容不同的新文档

### 技术栈

- **核心库**: python-docx (操作 DOCX 文件)
- **参考文献**: pyalex (OpenAlex API 客户端)
- **语言**: Python 3
- **编码格式**: UTF-8

### 关键技术特性

1. **格式保留**: 100% 保留原始格式(字体、字号、颜色、下划线、斜体等)
2. **嵌套表格支持**: 递归处理表格中的嵌套表格
3. **图片保留**: 自动识别并保留非文本元素(图片、公式等)
4. **域代码保护**: 识别并保护公式、交叉引用、目录等动态内容
5. **参考文献验证**: 使用 OpenAlex API 搜索真实学术文献,避免编造
