# 工作日志

## docx-essay-writer Skill 开发总结

### 项目概述

创建了独立的 `docx-essay-writer` skill，实现四阶段工作流：**模板分析 → AI 内容生成(Markdown) → 用户审阅 → 格式转换+验证**。所有脚本独立实现，不依赖其他 skill。

核心规则：参考文献禁止编造（Rule 0）、必须先分析模板（Rule 1）、章节结构与模板一一对应（Rule 2）、参考文献须用内置检索工具（Rule 3）。

### 模块清单

| 脚本 | 功能 |
|------|------|
| `analyze_template.py` | 解析 DOCX 模板：标题层级树、样式映射、页面布局、段落格式、特殊元素统计 |
| `search_references.py` | 调用 OpenAlex API 检索真实学术文献，输出 GB/T 7714-2015 格式，支持多主题去重 |
| `md_to_docx.py` | Markdown → DOCX 转换，样式/布局全部从模板动态读取 |
| `verify_format.py` | 对比输出与模板的格式一致性（页边距、页面大小、页眉页脚、样式文件、行距） |

### md_to_docx 核心能力

- **动态样式**：从 `styles.xml` 按名称查找 styleId，优先级链 `论文正文 → Body Text → Normal`
- **页面布局**：从模板 `sectPr` 直接复制，不硬编码
- **LaTeX → OMML**：递归解析 12 级优先级（函数括号 → 矩阵 → cases → 根号 → 分数 → 上下标 → 符号）
- **支持元素**：标题(h1-h3)、正文、加粗、行内/块级公式、表格、图片、有序/无序列表、代码块

### v2 重构：消除硬编码

**核心改动**：新增 `extract_template_formats()` 函数，从模板动态提取所有格式参数。

| 修复项 | 修复前(硬编码) | 修复后(动态提取) |
|--------|---------------|-----------------|
| 正文样式 | `Body Text` (id=5) | `论文正文` (id=29)，频率分析自动识别 |
| 标题 numPr | 固定 `ilvl='1', numId='0'` | 按模板各级标题样式读取，无则不添加 |
| 公式字体 | `eastAsia='宋体'` | 样式链追踪（style → link → basedOn） |
| 语言设置 | `en-US / zh-CN` | 从样式链动态提取 |
| 列表缩进 | `left/hanging='420'` | 从模板列表/正文样式提取 |
| 图片最大宽度 | `5274310` EMU | `pgSz.w - pgMar.left - pgMar.right` 动态计算 |
| font_hint | 全部 `'eastAsia'` | 从 `docDefaults` 推导 |

新增专用样式识别：参考文献(id=47)、图片(id=39)、三线表(id=46)。

### v3 修复：三线表支持

**问题**：表格使用全边框，而模板通过单元格级 `tcBorders` 实现三线表（表头上粗线+下细线，末行下粗线，其余无边框）。

**方案**：`extract_template_formats` 新增单元格级边框提取（header/body/last 三种模式），`_add_table_row` 按行位置应用对应边框、底纹、缩进、文字属性。

### 测试验证

三项测试均通过完整格式验证（页边距、页面大小、页眉页脚、styles.xml 完全匹配）：

1. **任务书**（v4 form-fill）：自动检测为 form 类型，直接修改模板 XML，格式完全一致
2. **绪论**（essay 模式）：Markdown → DOCX 转换，含 30 个 OMML 公式、7 个表格、10 篇真实参考文献
3. **任务书 v1**：XML 文本替换方式，表单式文档格式完全一致

### v4 重构：表单模板支持（form-fill 模式）

**问题**：模板无标题层级树（如任务书.docx）时，系统假设所有模板都有 heading 结构，导致输出格式完全不对。

**方案**：自动检测文档类型，新增 form-fill 模式——直接修改模板 XML 而非重新生成，100% 保留原始格式。

| 组件 | 修改 |
|------|------|
| `analyze_template.py` | 新增 `detect_document_type()` 返回 essay/form；新增 `extract_content_structure()` 提取编号分节、段落索引、样本内容、header/footer 区域；`analyze()` 输出增加 `document_type` 和 `content_structure` |
| `md_to_docx.py` | 新增 `_is_form_template()` 检测；新增 `_fill_template()` 表单填充核心逻辑；`convert()` 自动分流 essay/form 两种模式；新增 `_parse_md_sections()` / `_clean_form_content()` / `_match_section_label()` 辅助方法；注册 XML 命名空间前缀确保序列化一致 |
| `verify_format.py` | 修复 XML 自闭合标签空白比较假阳性（`/>` vs ` />`） |
| `SKILL.md` | Phase 1 输出增加 document_type 说明；Phase 2 拆分为 2A（essay）和 2B（form）两个分支；Phase 4 说明自动检测转换模式；新增表单示例 |

**form-fill 核心逻辑**：
1. 用 `extract_content_structure()` 识别模板的编号分节（`N、`模式）、段落索引、header/footer 边界
2. 用 `_parse_md_sections()` 将 Markdown 按 `## N、label` 拆分为段落组
3. 用 `_match_section_label()` 模糊匹配 Markdown 分节 → 模板分节
4. 对每个分节：保留模板段落的 `w:pPr`/`w:rPr` 格式，仅替换文本内容；段落数不同时 clone/remove
5. 通过 `_clean_form_content()` 过滤 `---`、Markdown 表头、`**bold**`、footer 等语法

**无硬编码**：分节检测基于正则模式 `^\d+[、.．]\S`，footer 检测基于通用模式 `^(备\s*注|指导教师|...)`，标签匹配基于归一化文本。

### v5 修复：表单模板 run 格式继承（下划线丢失）

**问题**：form-fill 模式创建新段落时，`_create_para_like` 和 `_replace_para_content_runs` 使用段落**最后一个 run** 的 `w:rPr` 作为格式参考。但某些段落（如"第一部分 收集资料...（2 周）"）最后一个 run 是尾部元数据（无下划线），而实际内容 run 有下划线，导致新段落丢失下划线格式。

**方案**：新增 `_find_content_rpr(para)` 方法，按文本长度加权找到最具代表性的内容 run 的 `rPr`。核心启发式：文本最长的 run 几乎总是代表主要内容区域的格式（标签前缀和尾部元数据通常较短）。

| 组件 | 修改 |
|------|------|
| `md_to_docx.py` | 新增 `_find_content_rpr()` 静态方法；`_replace_para_content_runs` 和 `_create_para_like` 改用 `_find_content_rpr` 替代 `_clone_run_rpr(runs[-1])` |

**无硬编码**：格式选择完全基于段落内 run 的文本长度分析，不依赖任何位置假设或固定规则。

**测试验证**：任务书模板格式验证全部通过（页边距、页面大小、页眉页脚、styles.xml 完全匹配），第4节内容段落下划线正确继承，essay 模式不受影响。

### v6 修复：表格单元格行内公式支持

**问题**：表格单元格中的行内公式（如 `$P_N$`）没有被正确转换为 OMML 格式，而是作为纯文本显示。

**原因**：`_add_table_row` 方法直接将单元格文本作为纯文本处理，没有像 `_body_para` 和 `_list_para` 那样使用正则 `re.split(r'(\$[^$]+\$)', text)` 分割并处理行内公式。

**方案**：
1. 新增 `_add_cell_rich_runs()` 方法，专门处理表格单元格中的富文本
2. 支持 `$inline formula$` 行内公式转换（调用 `latex_converter.convert()` 生成 OMML）
3. 支持 `**bold**` 和 `*italic*` 标记
4. 继承单元格模板格式（来自 `cell_rPr_tmpl` 的 bold/color/sz 等属性）
5. 保留行/列标题的粗体标志 `bold` 参数

**实现要点**：
- 公式部分直接追加 `m:oMath` 元素到段落
- 文本部分创建 `w:r` 元素并应用模板格式
- 方法签名：`def _add_cell_rich_runs(self, p, text, cell_rPr_tmpl=None, bold=False)`

**无硬编码**：公式转换复用现有的 `LatexToOmmlConverter`，格式提取沿用 `cell_rPr_tmpl` 模板系统。
